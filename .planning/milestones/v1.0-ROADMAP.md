# Milestone v1.0: OSS Core

**Status:** ✅ SHIPPED 2026-04-14

## Overview

DemoHunter v1.0 shipped the full local-first OSS core: scaffold flow, Playwright-native authoring DSL, two-pass generation engine, OpenAI-backed narration with deterministic cache behavior, portable `.demohunter/` output artifacts, and OSS adoption polish through examples, docs, skill docs, CI, and licensing.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Keep DemoHunter a thin wrapper on top of Playwright rather than a test runner or auth abstraction.
- Use a two-pass generation model so recording follows resolved narration timing instead of API latency.
- Treat `.demohunter/<tour-id>/` as the stable OSS-to-cloud handoff boundary.
- Make `video.mp4` the baseline portable artifact, with `video.webm` explicitly opt-in.
- Keep live OpenAI narration out of default CI so the public verify path stays secret-free.

**Issues Resolved:**

- Replaced the Phase 1 smoke path with a real two-pass generator.
- Added deterministic narration caching with offline reuse and corrupt-cache recovery.
- Locked a strict manifest/checksum contract for portable output.
- Verified source CLI, built CLI, and example-app consumer roots against the same product path.
- Shipped public onboarding docs, skill docs, CI, and MIT licensing for OSS launch.

**Issues Deferred:**

- Hosted watch pages, cloud upload, managed TTS, and sharing remain deferred to the next milestone.
- Built-in playback UI, browser editing, plugin infrastructure, and unscripted recording remain out of scope.

**Technical Debt Incurred:**

- `05-VALIDATION.md` is still stale relative to the restored `05-VERIFICATION.md`.
- Milestone helper state can briefly outrun tracker updates until archival docs are written.
