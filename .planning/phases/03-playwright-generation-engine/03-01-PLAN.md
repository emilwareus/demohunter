<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authored tour -> pass-1 collector | User-authored Playwright code emits runtime events that become generator replay input |
| loaded config -> page navigation | `baseURL` is untrusted input that controls which target Playwright opens |
| runtime event stream -> timeline contract | Event ordering and payload shape drive all later replay behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | T | pass-1 timeline contract | mitigate | Centralize runtime-event and timeline types in `generator-types.ts` so pass 2 consumes the same strongly typed contract emitted by pass 1. |
| T-03-01-02 | D | baseURL navigation | mitigate | Use direct `page.goto(new URL(config.baseURL).href)` and surface Playwright failures directly instead of masking them with retries or custom readiness automation per D-01 through D-03. |
| T-03-01-03 | R | runtime event ordering | mitigate | Add collector tests that assert exact lifecycle and event ordering so replay inputs cannot silently drift from the Phase 2 runtime surface. |
</threat_model>
