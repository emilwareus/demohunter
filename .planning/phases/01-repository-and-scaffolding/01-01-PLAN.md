<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo -> package manager | Third-party packages and workspace scripts are pulled into a new local toolchain |
| root config -> package builds | Shared compiler settings affect every package in the workspace |
| executable bins -> developer shell | Stub binaries become command entrypoints and must not run unsafe behavior yet |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | T | root `package.json` dependencies | mitigate | Keep Phase 1 package manifests minimal; do not add OpenAI, Zod, ffmpeg, or other unused runtime dependencies in this plan. |
| T-01-01-02 | E | CLI bin stubs | mitigate | Make both bin files explicit no-op stubs that print a fixed message and exit non-zero until Plan 03 implements real commands. |
| T-01-01-03 | T | shared TypeScript config | mitigate | Use explicit `NodeNext`/ESM settings in checked-in tsconfig files rather than ad hoc per-package compiler options. |
| T-01-01-04 | R | placeholder examples/skills | accept | Placeholder README-only directories carry no execution path and do not expose user data or credentials. |
</threat_model>
