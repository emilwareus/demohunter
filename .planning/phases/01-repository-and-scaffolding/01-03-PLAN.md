<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> scaffold filesystem writes | User-supplied command invocations control file creation inside an existing repo |
| tour file/config -> Playwright execution | Imported local code drives browser automation during smoke generation |
| output directory -> generated artifacts | CLI writes files under `.demohunter/` and must avoid unsafe overwrite behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | T | scaffold copy targets | mitigate | Copy only the fixed starter paths and reject overwrite with `Refusing to overwrite existing file: <path>` unless `--force` is passed. |
| T-01-03-02 | T | generate output paths | mitigate | Write smoke artifacts only under `config.outputDir/.demohunter/<tour.id>/` and derive file paths with `path.resolve` from the loaded config/project root. |
| T-01-03-03 | E | imported config/tour modules | mitigate | Limit imported files to `demohunter.config.ts` in cwd and the explicit CLI-provided tour path; do not search arbitrary directories or execute shell commands. |
| T-01-03-04 | I | sample scaffold safety | mitigate | Keep the sample page local-only, do not embed credentials, and do not emit cloud/network setup in `init` or `generate`. |
| T-01-03-05 | D | Playwright smoke flow | accept | A local browser launch may fail on a misconfigured machine; the e2e test documents the required local runtime but this is not a security blocker for Phase 1. |
</threat_model>
