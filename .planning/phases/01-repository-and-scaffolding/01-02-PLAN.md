<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo config file -> CLI runtime | Untrusted local TypeScript config is imported and merged into runtime defaults |
| template files -> user repository | Scaffold content becomes executable code inside an existing repo |
| starter page -> Playwright automation | The sample page defines the selectors and DOM events the smoke flow relies on |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | T | `load-config.ts` dynamic import | mitigate | Load only `${cwd}/demohunter.config.ts`, resolve with `pathToFileURL`, and reject missing files with a fixed error instead of searching arbitrary paths. |
| T-01-02-02 | I | starter template content | mitigate | Keep starter assets local-only and exclude `OPENAI_API_KEY`, cloud URLs, and credentials from scaffolded files. |
| T-01-02-03 | T | config defaults | mitigate | Define defaults in one typed source file in `@demohunter/sdk` and import them into the CLI loader rather than duplicating values. |
| T-01-02-04 | E | starter tour/page behaviors | accept | The sample page is static local content with no network or credential access; no privileged actions are exposed in this plan. |
</threat_model>
