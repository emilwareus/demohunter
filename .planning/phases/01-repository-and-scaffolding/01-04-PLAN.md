<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> scaffold filesystem writes | User-supplied command invocations control file creation inside an existing repo |
| tour file/config -> Playwright execution | Imported local code drives browser automation during smoke generation |
| output root -> generated artifacts | CLI writes files under `config.outputDir` and must avoid unsafe overwrite behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | T | scaffold copy targets | mitigate | Copy only the fixed starter paths and reject overwrite with `Refusing to overwrite existing file: <path>` unless `--force` is passed; prove this with automated tests. |
| T-01-04-02 | T | generate output paths | mitigate | Write smoke artifacts only under `${config.outputDir}/${tour.id}/` and derive file paths with `path.resolve` / `path.join` from the loaded config and tour id. |
| T-01-04-03 | E | imported config/tour modules | mitigate | Limit imports to `demohunter.config.ts` in cwd and the explicit CLI-provided tour path; do not search arbitrary directories or execute shell commands. |
| T-01-04-04 | I | sample scaffold safety | mitigate | Keep the sample page local-only, do not embed credentials, and do not emit cloud/network setup in `init` or `generate`. |
| T-01-04-05 | D | Playwright smoke flow | accept | A local browser launch may fail on a misconfigured machine; the e2e test documents the required local runtime but this is not a security blocker for Phase 1. |
</threat_model>
